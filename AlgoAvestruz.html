Algoritmo del Avestruz

El algoritmo del Avestruz es una estrategia de resolución para problemas de interbloqueo en los sistemas operativos. Consiste en ignorar el interbloqueo, fingir que no existe. Este método es el más simple de implementar ante el problema de interbloqueos, ya que solamente no se actúa en contra del problema. Esta solución se aplica cuando el interbloqueo no representa un problema significativo para el funcionamiento del sistema.

Condiciones Optimas

Esta solución se vuelve la más óptima bajo las siguientes condiciones (Gonzáles Nalda, 2024):

  • Sucede con poca frecuencia.
  • Si ocurre no hay consecuencias graves.
  • El costo de solucionar el interbloqueo es alto en recursos.

En otros casos sin embargo causa un impacto con mayores desventajas para el sistema operativo.

Ventajas

Las ventajas de este algoritmo incluyen:

  • El algoritmo es simple en implementación y diseño.
  • Evita el uso de recursos extra del sistema operativo en métodos de detección y prevención.
  • En sistemas con poca posibilidad de interbloques graves es el método más óptimo.

Desventajas

Las desventajas de este algoritmo incluyen:

  • Tiene repercusiones impredecibles acerca de los interbloqueos no resueltos.
  • Posibilidad de fallos del sistema operativo completo.
  • Puede llevar a la inanición de procesos dentro del sistema operativo.
